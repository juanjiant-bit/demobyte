Optimization / audit pass A

Applied changes:
1) BytebeatGraph preview isolation
   - Added preview_evaluate() to BytebeatGraph and Glide
   - Added StateManager::evaluate_preview()
   - AudioEngine spread preview now uses evaluate_preview() for both segment endpoints
   - Goal: avoid mutating live graph feedback / LP / DC / smoothing state when computing the alternate spread channel

2) ISR load reduction for very slow modulation
   - Moved the ultra-slow snapshot drift / air sin() terms out of per-sample processing
   - Cached them once per control block (32 samples)
   - Kept a tiny phase-coupled operator term in the per-sample path so the voice does not become static

Why these were chosen first:
- They are coherent with the current architecture
- They reduce hidden instability without redesigning the whole engine
- They are low-risk relative to changing the signal identity of the instrument

Not changed yet:
- Full DSP-chain budget pass
- Sub oscillator trig path
- Cross-core parameter mailbox/barrier redesign
- Comb stereo split
